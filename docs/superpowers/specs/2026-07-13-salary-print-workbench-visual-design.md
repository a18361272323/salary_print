# 工资表工作台视觉与交互设计

## 方向

采用 A「精密档案室」：暖灰背景、墨绿主色、低对比纸张白与细网格，表达工资数据的可信、克制与可追溯。屏幕工作台可以有现代产品感；正式打印页保持纯净、无装饰。

## 信息结构

- 顶部：产品标识、当前状态和月份筛选。
- 筛选卡：薪资组、单月份日历选择器、生成预览与取消加载。
- 适配摘要卡：记录数、预计页数、纸张、字号和纸张升级建议。
- 主预览卡：物理打印页预览。
- 侧栏：可折叠的打印列配置。

窄屏时依次排列筛选、摘要、预览和列配置；打印时只输出物理打印页。

## 月份选择

- 使用单月份选择器，默认当前月；禁止自由输入。
- 控件展示格式为 `2026 年 07 月`，业务值固定为 `YYYYMM`。
- 可通过上/下月导航，不能选择未来月份。

## 组件规范

- Button：40px 高度、6px 圆角、细描边；主按钮墨绿实底，次按钮纸张白底。
- Card：12px 内边距、1px 半透明边框、柔和阴影、`backdrop-filter: blur()`；不堆叠大圆角。
- Input/Select：同一高度、明确焦点环、错误状态使用暗红而非高亮发光。
- Toast/提示：仅显示加载、适配失败、A3 建议、保存结果；不干扰预览。
- Empty State：说明“选择薪资组与所属期后生成预览”，不使用 Emoji。

## 动效与空间感

- 首次进入：顶部、筛选卡、摘要、预览按 40–80ms 节奏渐入。
- 悬停：按钮和卡片仅轻微抬升 1–2px，时长 160–220ms。
- 加载：使用进度条与数字，不使用大面积旋转动画。
- 背景：固定的 gradient mesh、极轻噪点和可选 Canvas 粒子；粒子默认低密度且在 `prefers-reduced-motion` 下关闭。
- 不使用 Three.js/React Three Fiber：该 srcdoc 是原生 JavaScript，且财务数据工作台不应让 3D 元素占据注意力。

## 技术边界

- 继续使用原生 JavaScript；不引入 React 和 Motion 运行时。
- 使用本地 CSS 设计令牌实现 Tailwind/shadcn 的视觉目标，保证 srcdoc 离线与 CSP 稳定性。
- 打印媒体查询移除所有玻璃、背景、动画和工作台 UI，保留表格、汇总与签字区。

## 验收标准

1. 用户只能以日历方式选择一个不晚于当前月的所属期。
2. 屏幕端有清晰的筛选、适配摘要、预览与列配置层级；打印端无装饰性背景。
3. 键盘焦点、错误状态、空状态和加载状态均可辨识。
4. 在 `prefers-reduced-motion: reduce` 下没有持续动画。
5. A4/A3 适配提示仍处于摘要卡，功能行为不受视觉改造影响。
